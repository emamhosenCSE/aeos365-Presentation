Strategy: Database-per-Tenant

SaaS deployment utilizes complete database isolation to ensure maximum security and flexibility for enterprise clients.

Data Sovereignty & Security

Complete data isolation. One tenant's data is physically separated from others, mitigating cross-contamination risks.

Performance Isolation

Heavy workloads from one tenant do not directly impact query performance of other tenants' databases.

Backup & Restore Flexibility

Granular backups allow restoring a single tenant's data to a specific point in time without affecting the entire platform.

Customization Potential

Enables schema modifications or specific extensions for premium enterprise tenants if required.

Standalone Mode Note

Self-hosted installations use a single database with no multi-tenancy overhead, simplifying on-premise maintenance.
