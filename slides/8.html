Dual Deployment Modes

aeos365 Project Proposal

Shared Codebase

SaaS Platform

aeos365 Cloud (Multi-Tenant)

Tenancy Model Multi-tenant (Subdomain ID)
Database Strategy Separate DB per Tenant
Infrastructure Vendor-Managed (Auto-Scaling)
Updates & Maintenance Automatic / Continuous
Billing Model Integrated Subscription (Stripe)
Access Domain tenant.aeos365.com

Standalone Product

Self-Hosted (Single-Tenant)

Tenancy Model Single-tenant (Dedicated)
Database Strategy Single Isolated Database
Infrastructure Customer-Managed (On-Premises/VPS)
Updates & Maintenance Manual / Package Manager
Billing Model License Key / External Invoice
Access Domain hrm.customer-domain.com

Both deployment modes are built from the same monorepo codebase, ensuring feature parity and seamless migration.
